Task: Planejar Release
No início de um projeto o time criará um Release Plan em alto nível.
Disciplines: Tarefas Scrum
Purpose
Para nivelar expectativas (internas e externas) e proativamente revelar necessidades e dependências.
Relationships
RolesPrimary Performer:
  • Scrum Team
Additional Performers:
  • ScrumMaster
InputsMandatory:
  • Product Backlog
    		Optional:
    • None
    Main Description

    No início de um projeto o time criará um Release Plan em alto nível. Não é possível que o time conheça tudo no início, portanto, um plano detalhado não é necessário. O Release Plan deverá abordar:

    • A quantidade e a duração dos Sprints
    • Quantas pessoas ou times deverão participar do projeto
    • O número de Releases
    • O valor a ser entregue em cada Release
    • A data de liberação dos Releases

    As principais informações para o Release Planning são:

    • A priorização do Product Backlog
    • A estimativa da velocidade
    • O Product Owner deve atender as datas importantes (time-to-market) impostas pelo mercado.
